# YAPU-CLI: LLM Installer & Onboarding Prompt

You are an AI coding assistant (Cursor, Cline, Windsurf, Antigravity, etc.). Your user wants you to install and initialize the **YapuCli** framework in this project to manage your context, your long-term memory, and your cognitive load efficiently.

## Task 1: Installation
Execute the following command in the terminal to install YapuCli globally:
```bash
npm install -g @davidsd/yapu-cli
```
*(If you lack global permissions, you can just use `npx yapu-cli` for the following steps).*

## Task 2: Initialization
Execute the initialization command at the root of this project:
```bash
yapu init
```
This will scaffold the memory ecosystem (the `.planning/` directory and the master files).

## Task 3: Rule Assimilation
Once initialized, read the `PROJECT.md`, `ROADMAP.md`, and `STATE.md` files generated in the project root. From this moment on, you **must strictly obey** the following directives:

1. **Continuous Memory:** Keep the `STATE.md` file constantly updated. Mark tasks with `[x]` when completed and `[/]` when you are working on them.
2. **Phase Archiving:** When you finish all tasks of the "Current Phase" in the `ROADMAP.md`, run `yapu archive` in the terminal to clear your short-term memory.
3. **Rescue (Auto-Heal):** If the user tells you to look at a CI/CD error, look for the `.planning/debug/RESCUE_*.md` file generated by Yapu and follow its instructions.
4. **Context GC:** If the user tells you to execute a `LORE_MASTER.md`, read it from `.planning/tasks/LORE_MASTER.md` and condense the massive context into `.planning/LORE.md`.

## Task 4: Agent Persistence
To guarantee you never forget these directives in future sessions, **create your system's corresponding rules file** in the project root (e.g., `.cursorrules` if you are Cursor, `.windsurfrules` if you are Windsurf, `.clinerules` if you are Cline).
The file content must be:
`Always read .planning/STATE.md and .planning/ROADMAP.md at the start of every task. Keep the state updated and check off tasks with [x]. If there is an error log, use yapu rescue.`

*(Special ecosystem note: If you are Antigravity or Antigravity-CLI, you have first-class support. Use your native artifacts system and the `yapu sync`/`yapu brain` commands to maintain total control of the project's deep memory).*

## Final Task: Report
Reply to your user confirming that **YapuCli has been installed**, that **you have created your rules file (.cursorrules/.windsurfrules/etc)**, and that you have assimilated the static memory rules. Advise them to open a secondary terminal and run `yapu dash` so they can see your progress in real time.
