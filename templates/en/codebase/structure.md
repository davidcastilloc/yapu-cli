## Usage Guide

**What to include:**
- Directory layout (ASCII tree for visualization)
- Purpose of each directory
- Location of key files (entry points, configs, core logic)
- Naming conventions
- Where to add new code (by type)
- Special/generated directories

**What NOT to include:**
- Conceptual architecture → that goes in ARCHITECTURE.md
- Tech stack → that goes in STACK.md
- Every individual file (focus on directories and key files)

**How to fill out this template:**
1. Use `tree -L 2` or similar to visualize structure
2. Identify top-level directories and their purposes
3. Note naming patterns by observing existing files
4. Locate entry points, configs, and main logic areas
5. Keep the tree concise (maximum 2-3 levels)

**Tree format (ASCII box-drawing characters):**
```
root/
├── dir1/           # Purpose
│   ├── subdir/    # Purpose
│   └── file.ts    # Purpose
├── dir2/          # Purpose
└── file.ts        # Purpose
```
