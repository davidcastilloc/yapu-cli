## Guía de Uso

**Qué incluir:**
- Layout de directorios (árbol ASCII para visualización)
- Propósito de cada directorio
- Ubicación de archivos clave (entry points, configs, core logic)
- Convenciones de nombres
- Dónde agregar código nuevo (por tipo)
- Directorios especiales/generados

**Qué NO incluir:**
- Arquitectura conceptual → eso va en ARCHITECTURE.md
- Stack tecnológico → eso va en STACK.md
- Cada archivo individual (enfocarse en directorios y archivos clave)

**Cómo llenar esta plantilla:**
1. Usar `tree -L 2` o similar para visualizar estructura
2. Identificar directorios top-level y sus propósitos
3. Notar patrones de nombres observando archivos existentes
4. Ubicar entry points, configs y áreas de lógica principal
5. Mantener el árbol conciso (máximo 2-3 niveles)

**Formato del árbol (caracteres ASCII box-drawing):**
```
root/
├── dir1/           # Propósito
│   ├── subdir/    # Propósito
│   └── file.ts    # Propósito
├── dir2/          # Propósito
└── file.ts        # Propósito
```
